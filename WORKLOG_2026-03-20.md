# 2026-03-20 Worklog

## Summary

Today was the first day the new native spawn and live counter work actually crossed the line from "partially working" into a more believable live runtime state.
The goal was to stop the spawn path from feeling fake or invisible, and to make the live counters finally reflect what the player actually did in the field.

## What Changed

- reworked the native spawn path so the risky direct keyboard-trigger execution no longer handles the full spawn stack inline
- hardened the spawned ped tracking path so spawned enemies are kept in tracked slots and can still be credited after they fall out of the immediate nearby-ped list
- widened kill attribution so it no longer depends only on `source_of_death == player_ped`
- got `AnimalKills`, `LootCount`, and `RewardEarned` reflecting in the live profile during real in-game testing
- got `EnemyKills` reflecting in the live profile during real spawned-combat testing
- replaced the bad player-model fallback path for spawned enemies with a more explicit NPC-candidate route and then a safer clone/visibility-oriented follow-up pass
- added stronger runtime logging around spawn requests, candidate selection, ped creation, and counter updates so future live bugs can be narrowed much faster
- kept the live game root on the newest build while iterating, instead of drifting into a stale install again
