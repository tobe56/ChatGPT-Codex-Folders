## Summary

Today shifted the Frontier Ledger native UI away from a Korean-first stabilization attempt and into an English-first rebuild baseline.
The goal was not final localization quality.

- changed the active in-game config baseline from `live_ko` to `live_en`

## Why This Reset Was Necessary

The recent Korean workbench failures showed that the active problem was no longer "just translation."
It was:

- mixed renderer paths
- layout tuned against Korean edge cases before the English baseline was locked
- long helper/detail/result text colliding with narrow modal layouts
- session/default dataset drift pushing the runtime back into Korean while debugging

The safest next step was to freeze the active baseline in English and make localization an explicit option seam instead of the current default runtime mode.

## Active English-First Decisions

- English is now the active development baseline
- Korean is still supported conceptually, but it is no longer the active default
- future Korean work must enter through the locale seam, not by re-breaking the primary renderer path
